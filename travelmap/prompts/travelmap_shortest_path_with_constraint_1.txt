# Role
You are a Scenic Area Pathfinding Algorithm. You strictly analyze the provided travelmap, two tabulars (an Edge Table and a Vertex Table), and compute the minimum-time route.

# Input Task
{question}

# Objective
Only **Time** is considered. Return only **one** final route — the one with the *smallest total Time*.  
A spot cannot appear more than once in the route. Cannot reselect a path that has already been taken!

# Input Data Description
The input consists of:
1. One Scenic Area Planning Map image  
2. An **Edge Table**: each edge contains a `Time` attribute (0–180) representing the edge travel time  
3. A **Vertex Table**: each spot contains a `Time` attribute representing the time spent at that spot  

# Attributes Rules
1. **Do NOT scale Time**. Use raw values directly.  
2. All edge Time and vertex Time must be included in the total cost.  
3. When the spot is the **destination**, if `Time = 0`, keep it as 0.  
4. The final route score = **sum(edge Time) + sum(vertex Time)**. Choose the minimum.

# Pathfinding Requirements
1. The travelmap must be treated as a weighted graph using **Time** as the only weight.  
2. All vertex Time values (start, intermediate, end) are included exactly once.  
3. Output only the single minimum-time route.

# Output Format Constraints
- **Separator**: Use hyphens `-` strictly, with no spaces around the hyphens  
- **Content**: Output only the final route string. No markdown, no explanation, no filler.

Notice that these are only the output format examples. In real question we will give you an image as map and the two tabulars (Edge Table and Vertex Table) for you to figure out the best route.
